# Versioning Notes

## v0.6.07 Change

Starting with v0.6.07, small incremental patches use zero-padded patch numbering.

## Reason

The project is still in prototype/foundation development.

Using normal minor jumps too quickly can make the project feel closer to 1.0 than it really is.

## Current Convention

```text
v0.6.07
v0.6.08
v0.6.09
v0.6.10
```

## Larger Version Jumps

Larger jumps should be reserved for meaningful milestones, such as:
- first playable vertical slice
- first complete save/load loop
- first full town economy loop
- first real dungeon/portal loop
- first public demo candidate
