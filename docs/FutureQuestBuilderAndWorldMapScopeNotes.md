# Future Quest Builder and World Map Scope Notes

## Long-Term Quest Direction

The current quest system is a placeholder.

The intended future direction is a structured notice/request builder.

## Example Structures

```text
Hunt [discovered monster] until [X killed]
Hunt [discovered monster] until [nest reduced to level Y]
Explore [known dungeon]
Scout [fogged zone]
Guard [route or town edge]
```

## Discovery Requirements

Quest options should depend on what the guild knows.

The player should not be able to request actions against unknown threats.

Potential sources of discovery:
- adventurer exploration
- scout reports
- fog of war reveal
- dungeon sightings
- monster nest discovery
- Guild Hall upgrades
- rumors/tavern/notice board systems later

## World Map Strategic Layer

The world map should eventually track:
- zones
- fog of war
- discovered monster types
- active nests
- nest level/growth
- dungeon/portal sites
- regional danger
- adventurer activity
- guild notices/quest pressure

## Critical Design Note

Adventurers should still have agency.

The guild/player should manage priorities through notices, rewards, and encouragement, not by directly puppeteering every action.
