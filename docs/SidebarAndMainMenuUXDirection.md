# Sidebar and Main Menu UX Direction

## Current Reality

The sidebar currently contains a lot of testing/debug information.

That is useful during development because it gives immediate feedback.

## Long-Term Direction

The final player-facing sidebar should be cleaner and easier to navigate.

Potential improvements:

```text
compact buttons
icons where useful
hover tooltips
collapsible sections
less raw debug text
more visual summaries
```

## ESC Main Menu Direction

Pressing ESC now opens the first placeholder main menu overlay.

Long-term, this should hold deeper controls that do not need to live in the sidebar.

Possible sections:

```text
Save / Load
Settings
Graphics
Audio
Controls
Accessibility
Quit / Return to Title
```

## Main Menu vs Sidebar

The sidebar should support fast management decisions.

The ESC menu should support deeper system controls.

## Why This Matters

The sidebar should not become a dumping ground for every detail.

Deeper information belongs in:
- full panels
- modal menus
- graphs
- reports
- tooltips
- ESC menu sections
