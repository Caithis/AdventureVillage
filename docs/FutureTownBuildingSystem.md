# Future Town-Building System

## Purpose

This document records the intended long-term direction for player-created town layouts.

This is not part of v0.1.2 implementation.

## Design Intention

The town should not be a fixed layout. The player should create the village themselves.

At game start:
1. The player begins with an open buildable area.
2. The tutorial guides the player to place the Guild Hall.
3. The player uses starting funds to place the Inn and General Store.
4. The player can later move buildings as the town grows.

## Buildable Content

Future build mode should support:
- Guild Hall
- Inn
- General Store
- Crafting buildings
- Housing
- Roads
- Decorations
- Utility structures
- Defensive structures, possibly later

## Land Expansion

The town map should start limited and expand over time.

Possible fiction:
- The player buys additional land plots from the kingdom.
- Some plots may need clearing before use.
- Expansion may unlock better layouts, defensive planning, and district specialization.

## Mechanical Goals

Building placement should eventually affect:
- Adventurer travel time.
- Shop accessibility.
- Town attractiveness.
- Worker efficiency.
- Road/path movement.
- Defensive layout during raids.
- Decoration/happiness bonuses.

## Deferred Until

Do not implement this until:
- Adventurer town routine works.
- Basic shopping works.
- World travel placeholder works.
- First combat/resource loop works.

The early fixed layout is a prototype stand-in, not the final town design.
