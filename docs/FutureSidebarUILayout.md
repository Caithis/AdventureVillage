# Future Sidebar UI Layout Direction

## Design Direction

The long-term UI should protect the center gameplay viewport.

The game should move toward:

```text
center: gameplay viewport / town map / world map
right sidebar: active menu or information panel
top or bottom bars: compact global information
```

## Sidebar Modes

The right sidebar can eventually host modes such as:

```text
Building Details
Build Menu
Debug Tools
Economy Trends
Adventurer Overview
Guild Hall Management
Quest / Notice Board
Market / Price Trends
```

## One Major Mode at a Time

Only one major sidebar mode should be open at a time when practical.

Example:

```text
Open Build Menu
→ closes Building Details panel

Open Debug
→ closes Build Menu panel

Open Economy Trends
→ closes Debug panel
```

This prevents the screen from becoming cluttered.

## Why This Fits the Game

Dungeon Frontier Guild-Town will eventually have many information layers:
- gold flow
- material stock
- shop demand
- adventurer traffic
- Inn occupancy
- service queues
- worker staffing
- monster pressure
- world map events
- price trends

A dedicated sidebar gives the player access to those systems without burying the town view under floating windows.

## Inspiration Direction

Management and strategy games often preserve the main map while presenting deep information in side panels. This is a good fit for the future UI.
